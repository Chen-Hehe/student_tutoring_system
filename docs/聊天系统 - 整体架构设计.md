# 🦞 聊天系统 - 整体架构设计

**项目：** 乡村助学平台（Student Tutoring System）  
**模块：** 实时聊天系统  
**技术栈：** Spring Boot + WebSocket + Redis + React

---

## 📋 系统概述

### 功能特性
- ✅ 实时消息推送（WebSocket + Redis Pub/Sub）
- ✅ 消息已读状态实时同步
- ✅ 聊天记录持久化（MySQL）
- ✅ 离线消息处理
- ✅ 心跳保活机制
- ✅ 自动重连机制

### 用户角色
- **学生端** - 发送消息、接收消息、查看已读状态
- **教师端** - 发送消息、接收消息、查看已读状态
- **家长端** - （待实现）查看孩子聊天记录
- **管理端** - （待实现）消息内容审核

## 📡 消息流转

### 场景 1：发送消息

```
学生 A                    后端                    教师 B
  |                        |                        |
  |--HTTP POST /api/chat/ |                        |
  |        send ---------->|                        |
  |                        | 1. 保存到数据库        |
  |                        |    is_read=false       |
  |                        |                        |
  |                        | 2. Redis Pub/Sub ----->|
  |                        |    chat:user:1001      | 3. WebSocket 推送
  |                        |                        |    收到消息
  |                        |                        |
  |<-----------------------|                        |
  |  返回 message_id       |                        |
```

### 场景 2：已读状态同步

```
学生 A                    后端                    教师 B
  |                        |                        |
  |                        | 1. 打开聊天窗口        |
  |                        |    调用 markAsRead     |
  |                        |                        |
  |                        | 2. 更新数据库          |
  |                        |    is_read=true        |
  |                        |                        |
  |<--WebSocket 推送-------|                        |
  |    type=read           |                        |
  |    更新 UI 显示✓         |                        |
```

## 🗄️ 数据库设计

### chat_records 表

```sql
CREATE TABLE chat_records (
    id BIGINT PRIMARY KEY,              -- 雪花算法 ID
    sender_id BIGINT NOT NULL,          -- 发送者 ID
    receiver_id BIGINT NOT NULL,        -- 接收者 ID
    message TEXT,                       -- 消息内容
    type INT DEFAULT 1,                 -- 类型：1=文字，2=图片，3=语音
    file_url VARCHAR(500),              -- 文件 URL（图片/语音）
    is_read BOOLEAN DEFAULT FALSE,      -- 已读状态 ⭐
    sent_at TIMESTAMP DEFAULT NOW(),    -- 发送时间
    deleted TINYINT DEFAULT 0,          -- 逻辑删除
    INDEX idx_sender_receiver (sender_id, receiver_id),
    INDEX idx_receiver_unread (receiver_id, is_read)
);
```

## 🔌 API 接口

```http

X-User-Id: 2001

{
  "senderId": 2001,
  "receiverId": 1001,
  "message": "你好",
  "type": 1
```

## 🛡️ 安全考虑

### 1. 身份认证
- WebSocket 连接时通过 `userId` 参数认证
- HTTP 接口通过 `X-User-Id` Header 认证
- 生产环境应使用 JWT Token

### 2. 权限控制
- 只能查看自己相关的聊天记录
- 只能标记自己收到的消息为已读
- 不能冒充其他用户发送消息

### 3. 数据验证
- 消息内容长度限制
- 文件类型白名单（图片/语音）
- SQL 注入防护（MyBatis 参数化查询）

## 📊 性能优化

### 1. 缓存策略
- 聊天记录 Redis 缓存 5 分钟
- 对话列表按需刷新
- 已读状态内存更新

### 2. 消息推送
- 仅推送给在线用户
- 离线用户不推送，下次拉取
- 批量标记已读，减少 DB 操作
