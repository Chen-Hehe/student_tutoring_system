# 匹配统计功能说明

## 功能概述

匹配统计功能用于展示师生匹配系统的整体数据统计，帮助管理员、教师和学生了解匹配情况。

---

## 核心指标

| 指标名称 | 说明 | 计算方式 |
|---------|------|---------|
| 总匹配数 | 系统中所有匹配记录的总数 | COUNT(*) |
| 已匹配成功 | 状态为"已匹配"的记录数 | status = 2 |
| 待确认 | 状态为"待确认"或"待家长确认"的记录数 | status IN (0, 1) |
| 已拒绝 | 状态为"已拒绝"的记录数 | status = 3 |
| 成功率 | 成功匹配占总匹配的比例 | 成功数 / 总数 × 100% |

---

## 匹配状态说明

| 状态码 | 状态名称 | 说明 |
|-------|---------|------|
| 0 | 待确认 | 匹配请求已创建，等待对方确认 |
| 1 | 待家长确认 | 学生和教师已确认，等待家长确认 |
| 2 | 已匹配 | 所有相关方已确认，匹配成功 |
| 3 | 已拒绝 | 任意一方拒绝，匹配失败 |

## 后端接口

### 接口地址

```
GET /api/matches/statistics
```

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| teacherId | Long | 否 | 教师 ID（教师视角统计） |
| studentId | Long | 否 | 学生 ID（学生视角统计） |

### 请求示例

**全局统计（管理员视角）：**
```bash
curl http://localhost:8080/api/matches/statistics
```

**教师个人统计：**
```bash
curl "http://localhost:8080/api/matches/statistics?teacherId=2041436700118394881"
```

**学生个人统计：**
```bash
curl "http://localhost:8080/api/matches/statistics?studentId=2041436713062017026"
```

### 响应格式

```json
{
  "code": 200,
  "message": "成功",
  "data": {
    "totalMatches": 12,
    "successfulMatches": 5,
    "pendingMatches": 5,
    "rejectedMatches": 2,
    "successRate": "41.67%",
    "teacherId": null,
    "studentId": null
  }
}
```

## 统计 SQL 示例

### 全局统计

```sql
SELECT 
    COUNT(*) AS totalMatches,
    SUM(CASE WHEN status = 2 THEN 1 ELSE 0 END) AS successfulMatches,
    SUM(CASE WHEN status IN (0, 1) THEN 1 ELSE 0 END) AS pendingMatches,
    SUM(CASE WHEN status = 3 THEN 1 ELSE 0 END) AS rejectedMatches,
    CONCAT(ROUND(SUM(CASE WHEN status = 2 THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 2), '%') AS successRate
FROM teacher_student_matches
WHERE deleted = 0;
```

### 按教师统计

```sql
SELECT 
    teacher_id,
    COUNT(*) AS totalMatches,
    SUM(CASE WHEN status = 2 THEN 1 ELSE 0 END) AS successfulMatches,
    SUM(CASE WHEN status IN (0, 1) THEN 1 ELSE 0 END) AS pendingMatches,
    SUM(CASE WHEN status = 3 THEN 1 ELSE 0 END) AS rejectedMatches,
    CONCAT(ROUND(SUM(CASE WHEN status = 2 THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 2), '%') AS successRate
FROM teacher_student_matches
WHERE deleted = 0
GROUP BY teacher_id;
```

---

## 注意事项

1. **数据精度**：成功率保留 2 位小数，使用百分比字符串格式
2. **分母保护**：当总匹配数为 0 时，成功率返回 "0.00%"
3. **逻辑删除**：统计查询包含 `deleted = 0` 条件
4. **外键约束**：`student_id` 和 `teacher_id` 分别引用 `students` 和 `teachers` 表
